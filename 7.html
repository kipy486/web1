8일차

1. 내 컴퓨터에 웹서버를 설치해 누구나 내 컴퓨터의 웹페이지를 보도록 해 보자.
2. 제대로 작동하는 이유는 한 가지지만, 작동하지 않는 이유는 수십가지가 넘는다.
3. 하나의 문제를 해결하는 과정에서 수많은 지식과 보답을 얻게 된다.
4. 웹서버 제품군: Apache, IIS, Nginx...
5. 검색: how to install apache http server 운영체제
6. 윈도우용 ; how to easy install apache on window -- Bitnami WAMP Stack
6. mac ; bitnami mamp stack (mac apache(server) mysql(db) php(middleware))
7. cmd+space>manager-osx>매니지먼트콘솔(apache관리도구)>manage servers> Start/Stop
8. finder>applications>mampstack>apache2>htdocs>index.html
9. http://127.0.0.1:8080/index.html
127.0.0.1 ip address(인터넷에 연결돼있는 각각의 컴퓨터의 고유한 주소)
8080 port(설치된 웹서버 구분)
server가 browser에게 해당 파일을 보내면 browser가 파일에 저장된 코드를 해석해서 표시하는 방식
10. file:///Applications/mampstack-7.3.13-0/apache2/htdocs/index.html
파일이 위치한 경로로, 직접 index.html파일을 읽어서 화면에 표시하는 방식
11. http: HyperText Transfer Protocol 웹페이지 전송을 위한 규약
12. 서로 다른 컴퓨터에 설치된 웹서버의 특정 디렉토리에 설치된 index.html을 웹브라우저가 자신의 컴퓨터에서 보는 것처럼 보게 해보자.
==>ip address가 필요하다. http://serversip.index.html (단 두 대 모두 같은 네트워크에 연결돼있어야 함)
(네트워크환경설정열기>고급>TCP/IP>IPv4주소)
